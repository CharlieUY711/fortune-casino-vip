# Estructura de Datos Supabase: Lógica de Apuestas Fortune Casino ## 1. Tablas del Sistema ### `profiles` (Perfiles de Usuario) Almacena la información central del jugador y su balance. * `id`: uuid (PK, vinculado a auth.users) * `username`: text (único) * `balance`: numeric (default: 0, no negativo) * `level`: text (ej: 'Bronce', 'Plata', 'Oro', 'Diamante Noir') * `is_approved`: boolean (control del SuperAdmin) * `avatar_url`: text ### `bets` (Registro de Apuestas) Historial detallado de cada jugada para auditoría y transparencia. * `id`: uuid (PK) * `user_id`: uuid (FK -> profiles) * `game_type`: text (ej: 'ruleta_noir', 'blackjack_vip', 'slots_sphinx') * `amount`: numeric (monto apostado) * `payout`: numeric (monto ganado, 0 si pierde) * `status`: text ('pending', 'completed', 'cancelled') * `created_at`: timestamp ### `transactions` (Movimientos de Créditos) Control de ingresos/egresos manuales realizados por el SuperAdmin. * `id`: uuid (PK) * `user_id`: uuid (FK -> profiles) * `amount`: numeric * `type`: text ('deposit', 'withdrawal', 'bonus') * `description`: text * `created_at`: timestamp ## 2. Funciones de Base de Datos (RPC) Para garantizar la integridad, usaremos funciones del lado del servidor que aseguren operaciones atómicas. ### `process_bet(p_user_id, p_amount, p_game_type)` 1. Verifica si el usuario tiene balance suficiente. 2. Descuenta el monto de la apuesta del balance en `profiles`. 3. Crea un registro en la tabla `bets` con estado 'pending'. 4. Retorna el `bet_id`. ### `settle_bet(p_bet_id, p_payout)` 1. Actualiza el registro en `bets` con el `payout` y estado 'completed'. 2. Suma el `payout` al balance del usuario en `profiles`. ## 3. Seguridad y Políticas (RLS) * **Usuarios:** Solo pueden leer su propio balance y su propio historial de apuestas. * **SuperAdmin:** Tiene acceso total de lectura y escritura en todas las tablas para gestionar los 12 perfiles. * **Backend:** Las apuestas solo pueden ser modificadas a través de las funciones RPC para evitar manipulaciones externas del saldo. ## 4. Próximos Pasos * Implementar disparadores (Triggers) para actualizar el nivel de lealtad automáticamente según el volumen de apuestas. * Configurar suscripciones Realtime para que el balance se actualice instantáneamente en la interfaz del usuario tras una ganancia.
