# Guía de Ejecución Local: ¿Por qué no usamos 'npm run dev'? En este proyecto estamos utilizando **HTML5, Tailwind CSS (via CDN) y JavaScript puro**. Esto significa que: 1. **No hay dependencias de Node.js:** No necesitas instalar `node_modules` ni configurar un `package.json`. 2. **Archivos Estáticos:** Tu casino es una colección de archivos que el navegador puede leer directamente. 3. **Simplicidad:** Es mucho más rápido para la fase de **TESTING** porque lo que ves en tu carpeta es exactamente lo que subes a GitHub. ### Cómo ejecutarlo correctamente: #### Opción A: VS Code + Live Server (Recomendado) 1. Abre tu carpeta `fortune-casino-vip` en VS Code. 2. Haz clic derecho en `index.html`. 3. Selecciona **"Open with Live Server"**. 4. Tu navegador se abrirá en `http://127.0.0.1:5500/index.html`. #### Opción B: Python (Si tienes Python instalado) Ejecuta esto en tu terminal: ```powershell python -m http.server 8080 ``` Luego entra a `localhost:8080` en tu navegador. #### Opción C: Doble Clic Puedes simplemente hacer doble clic en `index.html`, pero **cuidado**: algunas funciones de Supabase o las rutas de navegación entre archivos podrían fallar por restricciones de seguridad del navegador (CORS) al usar el protocolo `file://`. Por eso siempre es mejor usar un servidor local pequeño.
