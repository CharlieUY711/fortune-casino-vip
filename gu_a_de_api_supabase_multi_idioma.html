# Documentación de API Supabase: Fortune Casino VIP ## 1. Gestión de Saldo y Billetera (PayPal/Binance) ### `get_user_balance(user_id)` Obtiene el saldo actual de dinero ficticio del perfil del usuario. * **Endpoint:** `profiles` table. * **Lógica:** Suscripción Realtime para actualizar la UI instantáneamente tras un depósito o premio. ### `request_transaction(user_id, amount, method, type)` Registra una solicitud de depósito o retiro. * **Métodos:** `'paypal'`, `'binance_usdt'`. * **Tipos:** `'deposit'`, `'withdrawal'`. * **Estado inicial:** `'pending'`. Requiere aprobación manual en el Panel de Admin. ## 2. Lógica de Apuestas en Juegos ### `place_bet(user_id, bet_data)` Llamada central para Ruletas, Blackjack, Poker y Slots. * **RPC Function:** `process_bet_v1`. * **Seguridad:** Verifica saldo > apuesta antes de insertar en la tabla `bets`. * **Campos:** `game_id`, `bet_amount`, `bet_selection` (ej: 'rojo', 'pleno 26'). ### `resolve_bet(bet_id, multiplier)` Cierra la apuesta y actualiza el saldo del usuario. * **RPC Function:** `settle_bet_v1`. * **Cálculo:** `payout = bet_amount * multiplier`. ## 3. Control de Acceso (Los 12 Invitados) ### `validate_invite_code(code)` Verifica si el código de invitación es válido y no ha sido reclamado. * **Endpoint:** `invitations` table. ### `approve_user_access(target_user_id)` Acción del SuperAdmin para activar una cuenta tras el registro. * **Lógica:** Cambia `is_approved` a `true` y asigna el bono inicial de bienvenida. --- *Este documento sirve como referencia técnica para la integración del frontend con el backend de Supabase en fortune.core.com.uy.*
